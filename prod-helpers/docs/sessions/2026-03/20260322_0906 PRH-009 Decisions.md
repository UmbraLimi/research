# Conv Decisions - 2026-03-22

## 1. Hyphenated Prefix Format for Conv Labels
**Type:** Strategic
**Topics:** cc-workflow, docs-infra

**Trigger:** PLAN.md item to update r-* skills to use PROJECT.yaml prefix in conv numbering and commit messages.

**Options Considered:**
1. `PRH-009` (hyphenated) ← Chosen
2. `PRH 009` (space-separated)
3. `PRH/009` (slash)

**Decision:** Use hyphenated format `PRH-009` for all conv labels: commit messages, headers, session doc filenames.

**Rationale:** Hyphen is a natural separator that works in filenames, commit messages, and display. Space-separated would cause issues in filenames. Slash could conflict with path separators.

**Consequences:** All 8 r-* skills updated. Commit messages now `PRH-009: title`. Session files now `20260322_0906 PRH-009 Dev.md`. Headers show `PRH-009 · MacMiniM4`.

## 2. .conv-current Stays as Pure Sequential Counter
**Type:** Strategic
**Topics:** cc-workflow, skills

**Trigger:** Initial proposal to store `PRH-009` in `.conv-current`.

**Options Considered:**
1. Store composed label `PRH-009` in `.conv-current`
2. Keep `.conv-current` as pure `009`, compose label at point of use ← Chosen

**Decision:** `.conv-current` remains a 3-digit zero-padded integer string. The prefix is read from `PROJECT.yaml` and composed with the counter at each skill's point of use.

**Rationale:** User identified that the conv number is a sequential system that auto-increments. The prefix is a labeling concern. Mixing them conflates counter mechanics with display formatting.

**Consequences:** Each skill that needs the full label has a `!`-backtick pre-computed `grep '^prefix:' PROJECT.yaml` line. No changes to conv-read-counter.sh or conv-read-current.sh scripts.

## 3. Apply Prefixed Conv to Session Doc Filenames
**Type:** Implementation
**Topics:** docs-infra, cc-workflow

**Trigger:** Question about whether prefix applies only to commits/headers or also to session doc filenames.

**Options Considered:**
1. Prefix in commit messages and headers only
2. Prefix in session doc filenames too ← Chosen

**Decision:** Session doc filenames include the prefixed conv: `{TIMESTAMP} {PREFIX}-{CONV} Dev.md` (e.g., `20260322_0906 PRH-009 Dev.md`).

**Rationale:** Makes session files self-identifying by project when viewed outside their directory context. Consistent labeling across all artifacts.

**Consequences:** r-dump and r-learn-decide skills updated with new filename format. r-eos passes timestamp args; each sub-skill reads its own prefix+conv from pre-computed context.
