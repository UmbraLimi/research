# Session Learnings - 2026-03-21

## 1. Conv numbering needs project scope in a monorepo
**Topics:** workflow

**Context:** prod-helpers has a CONV-COUNTER but lives inside the shared research/ repo. If a second project adds its own counter, they'd collide on pull/push.

**Learning:** Per-project conv numbers with a registered prefix (e.g. PRH-004) solve both collision and commit-message ambiguity. The prefix registry at repo root prevents duplicates.

---

## 2. Name files for future growth, not current content
**Topics:** workflow

**Context:** Initially created files as PROJECT-PREFIX.yaml / PROJECT-PREFIXES.yaml. Immediately renamed to PROJECT.yaml / PROJECTS.yaml because the prefix is just the first field — more metadata will follow.

**Learning:** When a YAML file will accumulate fields over time, name it for the entity (PROJECT), not the first field (PREFIX). Saves a rename later.

---

## 3. Skills with missing scripts still load but fail at runtime
**Topics:** skills

**Context:** /r-end and /r-learn-decide reference `.claude/scripts/*.sh` via `!` backtick interpolation. The scripts don't exist yet, causing skill load failures.

**Learning:** Skills that depend on scripts need either the scripts created alongside them, or graceful fallback. Currently this is a known gap — scripts were deferred (Decision #5 in DECISIONS.md mentions wrapper scripts but they haven't all been built).
