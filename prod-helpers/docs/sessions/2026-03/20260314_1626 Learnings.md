# Session Learnings - 2026-03-14

## 1. Append-only state files need a consolidation lifecycle
**Topics:** skills, workflow

**Context:** Adding append mode to `/r-save-state` so multiple conv states can stack in RESUME-STATE.md without overwriting.

**Learning:** Append alone creates unbounded growth. The append feature only works as part of a closed loop: save appends (max 2 blocks) → resume consolidates back to 1 → next save can append again. Without the consolidation step in `/r-resume`, a 3rd, 4th, etc. block would accumulate and the file would become unwieldy and ambiguous about what's current.

---

## 2. Consolidation must explain reasoning with evidence
**Topics:** skills, workflow, agents

**Context:** When `/r-resume` consolidates multi-block RESUME-STATE.md, the initial design just showed groupings (done / interactions / remaining) and asked "rewrite? y/n".

**Learning:** The user needs to see *why* each item was classified — "done because file X exists", "pending because no commits touch this", "interacts because both blocks reference the same skill file". Without evidence, the user can't catch misclassifications. This is an instance of the broader "ask before deciding" principle: don't just present conclusions, show the reasoning so the user can validate.
