# Conv Decisions - 2026-03-21

## 1. Bulk-Sync All r-* Skills from peerloop-docs
**Type:** Strategic
**Topics:** skills, cc-workflow

**Trigger:** Comparison revealed peerloop-docs had accumulated many improvements (dirty-repo guard, structured commits, conv terminology, insight capture, TodoWrite-all-gaps, PLAYBOOK.md routing) that prod-helpers lacked.

**Options Considered:**
1. Cherry-pick only critical fixes ← Conservative
2. Full sync of all improvements ← Chosen
3. Wait until skills stabilize before syncing

**Decision:** Port all improvements from peerloop-docs to prod-helpers in one pass, then treat prod-helpers as the self-contained reference going forward.

**Rationale:** The skills are actively being used and the gap was widening. A single bulk sync now avoids piecemeal drift. The user explicitly said to do a full update and stop cross-checking.

**Consequences:** All 10 r-* skills rewritten. CLAUDE.md updated. Memory entry created to monitor further drift during stabilization period.

> **Insight:** When two projects share a skill framework, designating one as "upstream" and syncing periodically is cleaner than ad-hoc porting. But the sync must be environment-aware — runtime differences (single vs dual repo, env var availability) mean not everything can be copied verbatim.

## 2. `!` Backtick Paths: Relative in SKILL.md, Resolved in Scripts
**Type:** Implementation
**Topics:** cc-workflow, skills

**Trigger:** $CLAUDE_PROJECT_DIR paths from peerloop-docs broke immediately when ported to prod-helpers `!` backtick lines.

**Options Considered:**
1. Revert all to relative paths ← Chosen (with internal script resolution)
2. Keep $CLAUDE_PROJECT_DIR with fallback syntax in SKILL.md
3. Investigate why the env var isn't set and fix it

**Decision:** Use relative paths in `!` backtick lines (`.claude/scripts/foo.sh`). Scripts internally resolve via `${CLAUDE_PROJECT_DIR:-$(pwd)}` for portability.

**Rationale:** This is the proven pattern from peerloop-docs Conv 003. The `!` backtick environment doesn't inherit hook env vars. Relative paths work because skills execute from the project root.

**Consequences:** All 10 skills updated. Pattern documented in memory for future skill development.

## 3. Adopt "Conv" Terminology Over "Session"
**Type:** Implementation
**Topics:** cc-workflow, docs-infra

**Trigger:** peerloop-docs had already renamed "session" → "conv" everywhere. prod-helpers still used "session".

**Options Considered:**
1. Keep "session" ← Original
2. Adopt "conv" ← Chosen (matches peerloop-docs)

**Decision:** Rename all "session" references to "conv" in skill names, descriptions, headings, and CLAUDE.md.

**Rationale:** Consistency across projects. "Conv" is more precise — a conversation maps 1:1 to a `/r-start` → `/r-end` cycle, while "session" is ambiguous.

**Consequences:** Updated in r-eos, r-end, r-dump, r-learn-decide, r-save-state, r-docs, and CLAUDE.md.

**Type:** Implementation
**Topics:** cc-workflow, docs-infra

**Trigger:** peerloop-docs routes workflow/convention decisions to PLAYBOOK.md separately from code decisions in DECISIONS.md.
