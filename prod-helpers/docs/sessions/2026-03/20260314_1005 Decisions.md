# Session Decisions - 2026-03-14

## 1. Use Wrapper Scripts for Piped `!` Backtick Commands
**Type:** Implementation
**Topics:** skills, workflow

**Trigger:** Skills with piped `!` backtick commands failed the permission checker.

**Options Considered:**
1. Remove `!` backticks and use tool-based alternatives (Read, Grep)
2. Rewrite pipes as single commands (e.g., `grep -m` instead of `grep | head`)
3. Pre-approve exact full command strings in settings (brittle)
4. Wrap pipelines in shell scripts in `.claude/scripts/` ← Chosen

**Decision:** Extract piped commands into named shell scripts under `.claude/scripts/`, called via `!` backtick. One blanket permission rule `Bash(.claude/scripts/*)` covers all.

**Rationale:** Preserves `!` backtick build-time determinism, provides self-documenting filenames in skills, single permission rule covers current and future scripts, and the pattern is already proven in peerloop-docs.

**Consequences:** Created 8 scripts, updated 6 skills. Added `Bash(.claude/scripts/*)` to settings.

## 2. Keep Broad Bash Permission Patterns as Safety Net
**Type:** Implementation
**Topics:** skills, workflow

**Trigger:** Discovered that `Bash(sed:*)`, `Bash(grep:*)` etc. do work for piped commands once settings are loaded.

**Options Considered:**
1. Remove broad patterns since scripts handle everything now
2. Keep broad patterns alongside scripts ← Chosen

**Decision:** Keep `Bash(sed:*)`, `Bash(cat:*)`, `Bash(head:*)`, `Bash(grep:*)` in settings with `Bash(sed -i:*)` in deny list.

**Rationale:** Safety net for future inline pipes in skills or ad-hoc use. The deny rule on `sed -i` preserves the approval prompt for in-place edits.

**Consequences:** Settings have both script and broad command patterns.

## 3. CLAUDE.md: Preserve `!` Backtick Determinism and Present Durable Options
**Type:** Strategic
**Topics:** workflow

**Trigger:** User feedback that simplest fixes were being acted on without presenting more durable alternatives.

**Options Considered:**
1. Continue with implicit approach (present one fix)
2. Add explicit guidance to CLAUDE.md ← Chosen

**Decision:** Added two sections to CLAUDE.md: (1) never replace `!` backtick pre-computed context without user approval, (2) always present simplest and most durable options, never act on simplest without approval.

**Rationale:** Early-stage project where patterns are forming — quick fixes can lock in fragile conventions. User wants informed trade-off decisions.

**Consequences:** Two new sections in CLAUDE.md, feedback memory saved for cross-project awareness.
