# Conv Learnings - 2026-03-22

## 1. Obsidian CLI Requires Manual Registration Per Machine
**Topics:** obsidian, credentials

**Context:** Checking Obsidian CLI availability on MacMiniM4 during API token setup.

**Learning:** Even with Obsidian Desktop installed, the CLI is not available until the user manually enables it via Settings → General → Command line interface → "Register CLI". This adds the binary to PATH via `~/.zprofile`. Each machine needs this done independently. After registration, `source ~/.zprofile` is needed in the current shell.

**Pattern:** Verify CLI availability with `which obsidian && obsidian version` after registration.

## 2. Joplin Web Clipper Ping Endpoint for Token Verification
**Topics:** joplin, credentials

**Context:** Needed to verify the Joplin API token was valid after storing it.

**Learning:** Joplin's Web Clipper API has a simple `/ping` endpoint at `http://localhost:41184/ping?token=...` that returns `JoplinClipperServer` on success. This is the lightest way to verify connectivity and token validity without touching any data.

---

## 3. Dynalist API Token Verification via file/list
**Topics:** dynalist, credentials

**Context:** Verifying the Dynalist token after storing it in credentials.

**Learning:** POST to `https://dynalist.io/api/v1/file/list` with `{"token":"..."}` and check `_code` field — returns `"Ok"` on success. Minimal data transfer since it only returns the file tree structure.

---

## 4. Conv Counter vs Project Prefix: Separation of Concerns
**Topics:** cc-workflow, skills

**Context:** Implementing PROJECT.yaml prefix in r-* skills. Initially proposed storing `PRH-009` in `.conv-current`.

**Learning:** The user correctly identified that `.conv-current` is a sequential counter (auto-incrementing integer) while `PRH-009` is a label composed at display time. Mixing labeling into the counter file would conflate two concerns. The counter stays pure (`009`), and each skill composes the label (`PRH-009`) from prefix + counter at point of use.

**Pattern:** When adding metadata to an identifier, compose at point of use rather than storing the composed form. This keeps the underlying data clean and the composition logic explicit.
