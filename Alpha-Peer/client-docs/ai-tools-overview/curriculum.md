# AI Tools Overview - Curriculum

## Course Structure

**Format:** 2 live 1-on-1 sessions (90 minutes each)
**Total Duration:** 3 hours + exploration time
**Delivery:** Interactive sessions via video call with hands-on demos
**Pace:** Beginner-friendly with focus on clarity over comprehensiveness

---

## Session 1: Text & Code AI (90 minutes)

### Module 1: Understanding AI Categories & Reducing Overwhelm (20 minutes)

**Learning Objectives:**
- Understand the major categories of AI tools
- Learn a framework for thinking about AI tools, not memorizing lists
- Develop strategies to avoid overwhelm and analysis paralysis
- Know where to focus your attention based on your needs

**Topics Covered:**
- **The AI Explosion:**
  - Why there are so many AI tools (low barriers to entry, API access)
  - Understanding the hype cycle
  - Separating signal from noise
  - How to stay informed without drowning
- **Major AI Tool Categories:**
  - **Text/Writing AI** - Content generation, conversation, analysis
  - **Code AI** - Programming assistance, debugging, learning
  - **Visual AI** - Images, videos, design
  - **Productivity AI** - Note-taking, meetings, research, scheduling
  - **Voice AI** - Speech-to-text, voice cloning, conversation
  - **Specialized AI** - Domain-specific tools (legal, medical, etc.)
- **Framework for Understanding:**
  - Input type (text, image, code, voice)
  - Output type (text, image, code, data)
  - Use case (creative, analytical, productive)
  - Skill level required (beginner, intermediate, advanced)
- **Reducing Overwhelm:**
  - You don't need every tool
  - Start with general-purpose tools
  - Add specialized tools as specific needs arise
  - Avoid "tool hoarding" - using tools just because they exist
  - Focus on workflows, not individual tools

**Hands-On Exercise:**
- Map your current tasks to AI tool categories
- Identify which categories are most relevant to your work
- Discuss which tools you've heard about and which matter to you

**Duration:** 20 min

### Module 2: AI Writing Tools (30 minutes)

**Learning Objectives:**
- Understand the major AI writing/text tools and their differences
- Learn what each tool does best and when to use it
- Gain hands-on experience prompting different AI models
- Develop intuition for choosing the right tool for writing tasks

**Topics Covered:**
- **Major Players:**
  - **ChatGPT** (OpenAI) - Most popular, general-purpose, large ecosystem
  - **Claude** (Anthropic) - Longer context, better at analysis, more nuanced
  - **Gemini** (Google) - Integrated with Google services, multimodal
  - **Perplexity** - Research-focused, cites sources, up-to-date information
  - **Others** - Grok, Llama, specialized models
- **Key Differences:**
  - Context windows (how much text they can process at once)
  - Strengths (creative writing vs analysis vs coding vs research)
  - Pricing models (free tiers, subscription costs)
  - Access methods (web, API, mobile app)
  - Special features (image understanding, internet access, etc.)
- **When to Use Which:**
  - **ChatGPT** - General tasks, brainstorming, quick questions, coding help
  - **Claude** - Long documents, nuanced analysis, complex reasoning, detailed writing
  - **Gemini** - Google Workspace integration, multimodal tasks
  - **Perplexity** - Research, fact-checking, current events
- **Common Use Cases:**
  - Writing assistance (emails, articles, marketing copy)
  - Brainstorming and ideation
  - Summarization and analysis
  - Learning and explanation
  - Translation and editing
  - Research and information gathering

**Hands-On Exercise:**
- Try the same prompt in ChatGPT and Claude
- Compare outputs and note differences
- Use Perplexity for a research question
- Discuss which tool felt better for which task
- Learn effective prompting basics

**Duration:** 30 min

### Module 3: AI Coding Tools (25 minutes)

**Learning Objectives:**
- Understand AI coding assistants and their capabilities
- Learn differences between major coding AI tools
- Know which tool matches your coding skill level
- See AI coding in action with hands-on demos

**Topics Covered:**
- **Major Coding AI Tools:**
  - **Claude Code** - Terminal-based, full project context, autonomous workflows
  - **Cursor** - VS Code fork, AI-powered IDE, inline generation
  - **GitHub Copilot** - Code completion, integrated into editors
  - **Windsurf** - Agentic coding, flow mode
  - **v0** (Vercel) - UI generation from text prompts
  - **Bolt.new** - Full-stack app generation in browser
- **Key Differences:**
  - **Interface** - Terminal vs IDE vs browser
  - **Autonomy** - Suggestions vs full file editing vs complete projects
  - **Context** - Single file vs full codebase awareness
  - **Skill level** - Beginners vs experienced developers
  - **Use cases** - Learning vs productivity vs prototyping
- **When to Use Which:**
  - **Claude Code** - Full projects, multi-file refactoring, learning to code (prerequisite: Intro to Claude Code course!)
  - **Cursor** - Daily coding, inline assistance, IDE lovers
  - **GitHub Copilot** - Code completion, pattern repetition, existing editor users
  - **v0/Bolt** - Quick prototypes, UI generation, no-code builders
- **What AI Coding Can Do:**
  - Generate boilerplate and repetitive code
  - Debug and fix errors
  - Explain code and concepts
  - Refactor and improve existing code
  - Build complete features from descriptions
  - Learn programming concepts
- **What AI Coding Can't Do (Yet):**
  - Replace deep software architecture thinking
  - Handle complex performance optimization reliably
  - Guarantee security without review
  - Understand business requirements without clear specs

**Hands-On Exercise:**
- Watch Claude Code demo (building a simple app)
- See Cursor inline code generation
- Compare coding AI approaches
- Discuss which tool matches your coding level
- Understand when to use coding AI vs learning manually

**Duration:** 25 min

### Module 4: Decision Framework - Choosing the Right Tool (15 minutes)

**Learning Objectives:**
- Develop a systematic approach to choosing AI tools
- Learn to evaluate tools based on your specific needs
- Avoid common tool selection mistakes
- Create your personal decision criteria

**Topics Covered:**
- **The Tool Selection Framework:**
  1. **Define the task** - What are you actually trying to do?
  2. **Match to category** - Which AI category fits this task?
  3. **Consider skill level** - What's your expertise in this area?
  4. **Evaluate options** - What are the 2-3 top tools in this category?
  5. **Check constraints** - Budget, privacy, integration requirements
  6. **Start simple** - Choose the most accessible option first
  7. **Upgrade later** - Switch only when you have a specific reason
- **Key Decision Criteria:**
  - **Cost** - Free tier, subscription, pay-per-use
  - **Privacy** - Data usage policies, enterprise vs consumer
  - **Integration** - Works with your existing tools and workflow
  - **Learning curve** - Time to productivity
  - **Context size** - How much information can it handle
  - **Output quality** - Best for your specific use case
- **Common Mistakes to Avoid:**
  - Subscribing to every tool "just in case"
  - Choosing based on hype instead of need
  - Not trying free tiers before paying
  - Ignoring privacy implications
  - Expecting one tool to do everything
  - Switching tools before learning the first one
- **Building Your Toolkit:**
  - Start with 1-2 general-purpose tools
  - Add specialized tools as specific needs arise
  - Reevaluate quarterly, not daily
  - Focus on mastery over collection

**Hands-On Exercise:**
- Apply framework to your actual needs
- Identify which tools you should prioritize
- Create a "start here" plan for AI adoption
- Discuss tool choices specific to your work

**Duration:** 15 min

## Session 2: Visual AI & Your Personal Toolkit (90 minutes)

### Module 5: AI Image & Video Generation (30 minutes)

**Learning Objectives:**
- Understand AI image generation tools and capabilities
- Learn about AI video generation (current state and limitations)
- See what's possible with visual AI through demos
- Know when visual AI is useful vs when traditional tools are better

**Topics Covered:**
- **AI Image Generation:**
  - **Midjourney** - Artistic, high quality, Discord-based, paid
  - **DALL-E** (OpenAI) - Accessible, ChatGPT integration, consistent
  - **Stable Diffusion** - Open source, customizable, requires technical setup
  - **Leonardo.ai** - Game assets, consistent characters
  - **Ideogram** - Good at text in images, photo-realistic
  - **Others** - Adobe Firefly, Canva AI, etc.
- **Key Differences:**
  - Image quality and style capabilities
  - Ease of use and learning curve
  - Pricing models (credits, subscriptions, free)
  - Control vs simplicity trade-offs
  - Commercial usage rights
- **Common Use Cases:**
  - Marketing visuals and social media
  - Concept art and ideation
  - Product mockups and prototypes
  - Illustrations for content
  - Photo editing and enhancement
  - UI/UX design inspiration
- **AI Video Generation:**
  - **Runway** - Professional editing, AI effects, gen-3 model
  - **Pika** - Simple interface, good for beginners
  - **Sora** (OpenAI) - Cutting edge, limited access
  - **HeyGen** - AI avatars, talking head videos
  - **Synthesia** - Professional AI video production
- **Current Limitations:**
  - Video quality not yet photorealistic (but improving rapidly)
  - Short clips (5-10 seconds typically)
  - Limited control over consistency
  - High computational cost
  - Still best for specific use cases, not general video production
- **When to Use AI vs Traditional Tools:**
  - Use AI for: Rapid ideation, concept exploration, unique styles
  - Use traditional for: Precision work, brand consistency, complex compositions
  - Hybrid approach often works best

**Hands-On Exercise:**
- Watch Midjourney image generation demo
- See prompting techniques for better results
- Explore AI video generation examples
- Discuss practical use cases for your work
- Learn effective prompting for visual AI

**Duration:** 30 min

**Learning Objectives:**
- Create a personalized AI toolkit based on your actual needs
- Learn to prioritize tools by value and frequency of use
- Develop a plan for learning and adopting new tools strategically
- Avoid tool overwhelm by focusing on what matters

**Topics Covered:**
- **Your AI Toolkit Strategy:**
  - **Tier 1 - Daily Drivers** (1-3 tools you use every day)
  - **Tier 2 - Weekly Tools** (3-5 tools for specific tasks)
  - **Tier 3 - Occasional** (Tools you know exist and can access when needed)
  - **Research List** (Interesting tools to explore later)
- **Matching Tools to Your Workflow:**
  - Map your common tasks to AI capabilities
  - Identify high-impact vs low-impact opportunities
  - Prioritize based on time saved and quality improved
  - Consider integration with existing tools
- **Learning Strategy:**
  - Master one tool before adding another
  - Start with free tiers and trial periods
  - Focus on one category at a time
  - Build competence before expanding
- **Budget Strategy:**
  - Identify which paid tools are worth it
  - Use free tiers strategically
  - Avoid subscription bloat
  - Calculate ROI based on time saved
- **Common Toolkit Examples:**
  - **Writer/Marketer:** Claude (writing), Midjourney (visuals), Perplexity (research)
  - **Developer:** Claude Code (coding), ChatGPT (quick queries), Cursor (IDE)
  - **Business Owner:** ChatGPT (general), Otter.ai (meetings), Notion AI (organization)
  - **Content Creator:** ChatGPT (scripts), Midjourney (thumbnails), Runway (video)
- **Integration and Workflow:**
  - How different AI tools work together
  - Automation between tools (n8n integration)
  - Consistency across tools
  - When to combine vs when to specialize

**Hands-On Exercise:**

### Module 8: Staying Current Without Overwhelm (20 minutes)

**Learning Objectives:**
- Develop strategies to stay informed about AI without drowning
- Learn how to evaluate new AI tools quickly
- Avoid FOMO and analysis paralysis
- Create a sustainable approach to AI tool discovery

**Topics Covered:**
- **The New Tool Evaluation Framework:**
  1. **Does it solve a problem I have?** (Not "is it cool?")
  2. **Is it better than my current solution?** (10x better, not 10% better)
  3. **What's the switching cost?** (Time, money, learning curve)
  4. **Is it mature enough?** (Beta vs production-ready)
  5. **Wait and see** vs **early adopter** decision
- **Staying Informed Strategically:**
  - Follow 2-3 quality AI newsletters (not 20)
  - Check AI news weekly, not hourly
  - Ignore most announcements (90% don't matter to you)
  - Focus on your use cases, not general hype
  - Trust expert curation over algorithm feeds
- **Avoiding FOMO:**
  - New doesn't always mean better
  - Tools you don't use don't help you
  - Missing a tool isn't missing out if it doesn't fit your needs
  - The best tool is the one you'll actually use
  - You can always adopt later when tools mature
- **The "Wait and See" Strategy:**
  - Let early adopters find the bugs
  - See which tools get traction vs fade
  - Join when value is proven, not promised
  - Your time is valuable - protect it
- **Quarterly Review Process:**
  - Every 3 months: Review your toolkit
  - What are you actually using?
  - What subscriptions can you cancel?
  - Are there new tools worth trying?
  - Has your workflow changed?
  - Update your Tier 1/2/3 lists
- **Resources Worth Following:**
  - Quality AI newsletters (Ben's Bites, The Rundown AI)
  - YouTube channels for demos and reviews
  - Reddit communities (r/artificial, r/ArtificialIntelligence)
  - Tool-specific communities for tools you use
  - Avoid: Hype-driven accounts, breathless "everything is changing" sources

**Hands-On Exercise:**
- Apply the new tool evaluation framework to recent AI announcements
- Create your "staying informed" strategy
- Plan your quarterly review process
- Identify which information sources to follow vs ignore
- Commit to avoiding tool FOMO

**Duration:** 20 min
